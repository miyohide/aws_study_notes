# Amazon Kinesis Data Streams vs Amazon Data Firehose

| 比較項目 | Amazon Kinesis Data Streams (KDS) | Amazon Data Firehose (ADF) |
| :--- | :--- | :--- |
| **主な用途** | **リアルタイム処理・カスタム分析**<br>ミリ秒単位の応答が必要なアプリ基盤 | **データロード・アーカイブ**<br>ストレージや分析基盤への自動転送 |
| **データ変換** | **コンシューマー側で実装**<br>読み取るアプリケーション側でロジックを記述 | **Lambda統合による自動変換**<br>配信過程で形式変換や加工が可能 |
| **配信・運用管理** | **シャードによる管理**<br>オンデマンドモードまたはプロビジョニングが必要 | **フルマネージド（サーバーレス）**<br>スケーリングやリトライもすべて自動 |
| **レイテンシ** | リアルタイム（通常200ms未満） | ニアリアルタイム（最小60秒または1MB） |
| **データの再利用** | **可能**<br>複数のコンシューマーが同じデータを読み取れる | **不可**<br>一度配信先へ送るとFirehose内には残らない |
| **主な宛先** | Lambda, KCL(EC2/ECS), Managed Service for Apache Flink | S3, Redshift, OpenSearch, Splunk, HTTP Endpointなど |

## ワークグループ

クエリやノートブックの実行環境を論理的に分離・管理する単位。使用するエンジンをApache Sparkに設定したいときもワークグループを使う。

## コスト最適化

クエリ結果の再利用を使う。以前に実行されたクエリと同一のSQLが再度実行された際に前回のクエリ結果をそのまま返すことでScanオペレーションを回避し、課金を大幅に抑制させる。

# 主要ストリーミングサービスの比較表

| 特徴 | **Kinesis Data Streams (KDS)** | **Amazon Data Firehose (ADF)** | **Managed Service for Apache Flink** | **Amazon MSK (Kafka)** |
| :--- | :--- | :--- | :--- | :--- |
| **主な用途** | 低遅延なリアルタイム処理 | 指定先へのデータ配信 (ETL) | 複雑なストリーム分析 | オープンソース互換の基盤 |
| **管理レベル** | プロビジョンド or オンデマンド | **完全マネージド (Serverless)** | 完全マネージド | マネージド (一部ノード管理) |
| **遅延 (Latency)** | **ミリ秒単位** | 60秒〜 (バッファリング依存) | ミリ秒単位 | ミリ秒単位 |
| **データ保持** | 最大365日間 | なし（配信のみ） | なし（処理のみ） | 長期保持可能 |
| **スケーリング** | シャード単位（手動/自動） | **フルオートスケール** | 処理能力に応じた自動制御 | ブローカー追加 |
| **実装負荷** | 高め (KCL/SDKの利用) | **低い (GUIで設定完結)** | 高め (Java/Python/SQL) | 高め (Kafkaエコシステム) |

# Lambda

## 予約済み vs プロビジョニング済み

| 項目 | 予約済み同時実行数 (Reserved) | プロビジョニング済み同時実行数 (Provisioned) |
| :--- | :--- | :--- |
| **主な目的** | **リソースの確保** と **実行制限（上限設定）** | **コールドスタートの抑制** (低レイテンシ化) |
| **動作** | 指定した数をその関数専用にキープする | 指定した数の実行環境を事前に初期化して待機させる |
| **スケーリング** | 設定値がそのまま実行数の**上限**になる | 設定値を超えてもオンデマンドでスケール可能* |
| **コスト** | **無料** | **有料** (設定した数・時間分課金) |
| **コールドスタート** | 発生する (環境は都度作成される) | **発生しない** (既に起動済みのため) |
| **設定対象** | 関数全体 | 関数の特定の「バージョン」または「エイリアス」 |
